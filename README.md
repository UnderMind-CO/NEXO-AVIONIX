# Project Management Workflow Documentation

## Agile/Scrum Framework

The Agile/Scrum framework is an iterative and incremental approach to software development that emphasizes flexibility, collaboration, and customer satisfaction. In our project, we adopt the following principles of Agile/Scrum:

1. **Sprints**: We work in time-boxed iterations called sprints, typically lasting 2-4 weeks.
2. **Daily Standups**: Team members meet daily to discuss progress, roadblocks, and next steps.
3. **Sprint Planning**: At the start of each sprint, we identify the work to be done and plan the tasks accordingly.
4. **Sprint Reviews**: At the end of each sprint, we review deliverables and gather feedback from stakeholders.
5. **Sprint Retrospectives**: We reflect on the previous sprint to identify improvements for future sprints.

## Sprint Planning

Sprint planning involves breaking down project goals into actionable tasks. This is done collectively in a team meeting at the beginning of each sprint. Tasks are prioritized based on the backlog and assigned to team members.

1. **Backlog Review**: Review product backlog items.
2. **Task Breakdown**: Break down high-priority items into tasks.
3. **Assignments**: Assign tasks based on team expertise and capacity.

## Time Estimations

We utilize story points to estimate the effort required for each task. The following scale is used:
- 1 Point: Very easy
- 2 Points: Easy
- 3 Points: Moderate
- 5 Points: Hard
- 8 Points: Very Hard

Estimations are discussed and consensus is reached among team members during sprint planning meetings.
